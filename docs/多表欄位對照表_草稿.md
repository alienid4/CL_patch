# 多工作表 — 欄位對照表（草稿，待確認）

> 目的：把 10 張數字表的欄位對到統一概念，讓同一套儀表板能自適應顯示。
> 狀態：**草稿**，需 USER 確認 E1–E4 與 Q4/Q6 後才開發。
> 來源：docs/弱點彙總報告(New)_V20260702 的TEST.xlsx（實際抽樣值）

## 通用規則（我打算這樣做）

- **嚴重度統一（Q4）**：一律映射到 Critical / High / Medium / Low / Unknown，並保留原字串（滑鼠停留可見）。
  - 英文：Critical/High/Medium/Low/Info → 同名（Info→Low 區）。
  - 中文：嚴重→Critical、高→High、中→Medium、低→Low。
  - BitSight 發現嚴重性：critical/severe→Critical、material→High、moderate→Medium、minor→Low。
  - BitSight Grade「WARN(Medium)」：取括號內等級。
  - N/A、空 → Unknown。
- **結案判定通則**：含「未結」或「進行中」= 未結案；含「結案/已結案/已修補」且無「未」= 已結案；其餘 = 其他。
  - 尾註如「(申請例外)」忽略，只看主狀態。
  - 無結案狀態欄的表（8）→ 有「結案日期」= 已結案，否則未結案。
- **真正到期日優先序**：例外核准期限 → 首次展延(上限/期限) → 修補期限 / 改善期限 / 預計完成日期（取該表最接近者）。
- **母體（Q2）**：不預設鎖單位；單位與結案狀態改成可篩選，含「全部」。

## 已確認（結論 2026-07-08）

- **E1（表3 嚴重度）**：採 **發現嚴重性 Finding Severity**（minor→Low、moderate→Medium、material→High、severe/critical→Critical）。
- **E2（表9 風險計數）**：**解析計數彙總**成嚴重度統計，「中*4 低*2」＝ 4×Medium＋2×Low，一列可貢獻多筆。
- **E3（表8 單位/負責人）**：**從欄位拆出**（分隔符 `-`）：單位＝前段、負責人＝後段（去括號註記）。
- **E4（結案判定通則）**：**採用**（未結/進行中＝未結案；結案/已結案/已修補且無「未」＝已結案；表8 用結案日期有值判定）。
- **Q4（嚴重度統一）**：**採用**中/英/BitSight → Critical/High/Medium/Low 映射，保留原值。
- **Q6**：**先各表各查**；「資安弱點追蹤窗口」表當參考、不做左側分頁。跨表全域搜尋列為未來項。

1　系統概述

1.1 架構

本系統為純前端單機應用程式，以瀏覽器開啟本機 HTML 檔案執行，不含應用程式伺服器、不含資料庫、不對外連線。

1.2 元件

元件說明對外連線
看板主程式HTML/JavaScript，於瀏覽器執行；解析 Excel 並產生統計與明細無
寄信小幫手
（選用，可停用）		本機 PowerShell 服務，供看板發送催辦信；查詢 AD 取得收件人、透過公司 relay 寄送僅公司內部 AD／SMTP relay

寄信小幫手屬選用模組，可於「其他功能 → 功能開關」關閉；關閉後系統為完全離線之檢視工具。

2　資料處理說明

2.1 資料分類

資料類別內容敏感等級
弱點掃描結果主機位址、弱點名稱、Plugin ID、嚴重度、修補期限機敏
權責資訊負責單位、負責人姓名內部
催辦紀錄寄送時間、收件人電子郵件、寄送結果內部（含個人資訊）
個人設定SMTP 主機、寄件人信箱、篩選偏好、小幫手權杖內部

2.2 資料流向

  • Excel 由使用者自行選取本機檔案，經瀏覽器 File API 讀入記憶體解析
  • 解析與統計全部於瀏覽器本機完成，無任何 HTTP 上傳行為
  • 唯一對外行為為催辦寄信：由本機小幫手查詢公司 AD、經公司內部 SMTP relay 寄送

2.3 儲存位置與保存

資料儲存位置保存與清除
匯入的 Excel 內容瀏覽器 localStorage（vulnDashboard.v2），以 Base64 編碼存放保留至使用者清除；「其他功能 → 清除暫存資料」可刪除
發信紀錄瀏覽器 localStorage，上限 500 筆（逾量自動汰舊）可於「發信紀錄」介面清空
發信稽核檔小幫手所在資料夾 mail_log.csv持續累積，由使用者自行保管與清理
存取權杖小幫手所在資料夾 agent_token.txt每次小幫手啟動即重新產生（舊權杖立即失效）
請注意：Base64 為編碼而非加密，localStorage 內容可由該電腦使用者讀取。 共用或公用電腦使用後，請務必執行「清除暫存資料」。

3　安全控制措施

依控制面向列示實作方式。「不適用」係因架構上不存在該風險面。

控制面向實作方式狀態
資料傳輸 看板不進行任何網路傳輸；催辦寄信經公司內部 relay，不經外部網路 已實作
對外暴露 寄信小幫手僅繫結 localhost，不接受外部主機連線 已實作
服務存取控制 小幫手每次啟動產生一次性權杖，寄信端點須帶正確權杖（X-Agent-Token）， 並限制請求來源（同源／本機）；驗證失敗回應 401 並拒絕執行 已實作
稽核軌跡 催辦寄送逐筆記錄時間、負責人、收件人、副本、結果與失敗原因； 同時寫入本機稽核檔，檔案被占用時自動改寫入備援檔並回報 已實作
錯誤處理 寄送失敗明確標示並告警（不以「已送出」概括）；操作逾時提示避免重複寄送； 寄送中鎖定按鈕防重複觸發 已實作
資料完整性 欄位無法辨識、狀態無法歸類、重複欄位名稱時主動警示並提供明細， 避免資料靜默遺失而未被察覺 已實作
輸出編碼 來源資料於畫面呈現前一律經 HTML escape，避免內容含標記字元造成注入或破版 已實作
機敏資訊管理 程式碼不內嵌任何主機位址、帳號、密碼或信箱；連線參數全由使用者於本機設定。 真實資料檔、權杖、催辦紀錄均排除於版本控制之外 已實作
身分驗證 單機工具無伺服器可驗證身分，存取控制邊界為作業系統帳戶（詳見 5.1）。 本工具不新增亦不降低既有防護 部署模式
靜態資料加密 本機暫存未加密，依賴作業系統帳戶隔離與磁碟加密政策； 提供清除功能（詳見 5.4） 需承擔
伺服器面控制 無伺服器、無資料庫、無對外服務埠 不適用

4　威脅情境與因應

威脅情境因應措施評估
弱點資料經網路外洩 系統無上傳行為、無外部連線，資料不離開本機 已控制
本機其他程式或網頁盜用寄信服務，冒用使用者名義發信 小幫手僅收本機連線；寄信端點須通過一次性權杖驗證與來源檢查 已控制
催辦未實際送達卻誤認為成功 修正失敗判定邏輯，失敗逐筆標示並告警；保留可查紀錄 已控制
資料被誤判或漏算，導致風險誤判 欄位／狀態無法辨識時主動警示；統計數字全數可下鑽核對明細 已控制
惡意內容夾帶於 Excel 欄位，於畫面觸發指令碼 所有輸出經 HTML escape；系統不執行來源資料內容 已控制
取得該電腦控制權者讀取本機暫存資料 應用層無法阻擋；依作業系統帳戶控管與磁碟加密政策。 惟該情境下攻擊者本即可直接開啟來源 Excel，非本工具新增之風險 需承擔
使用者自行轉寄含真實弱點之原始報告 屬使用行為，非系統可控範圍；以第 7 節使用者責任規範 使用行為

5　部署模式與安全邊界

評估基準：本工具是否使既有處境變差。單機工具的既有特性（無登入、資料存本機） 屬部署模式定義，非缺陷，故不列為風險項。

5.1 部署模式

單機本機工具。存取控制邊界為作業系統帳戶，與該電腦上的 Excel、Word 等本機檔案同一層級。

  • 本工具不新增身分驗證層——單機工具無伺服器可驗證身分，於前端實作之登入可被繞過， 提供虛假保護反而有害
  • 本工具亦不降低既有防護——能開啟看板者，本即可直接開啟同一份來源 Excel
  • 資料保護仰賴既有機制：Windows 帳戶隔離、資料夾權限、公司磁碟加密政策

5.2 本工具新增的風險面

相對於「電腦中原本就存有該份 Excel」，本工具實際新增之風險面為下列兩項，均已施控：

新增項目控制方式狀態
本機服務監聽連接埠
（寄信小幫手，選用模組）		 僅繫結 localhost 不對外；寄信端點須通過一次性權杖驗證與來源檢查； 權杖檔限縮為僅本人可讀；連接埠被占用時自動改用備用埠，並驗證回應方確為本服務 已控制
瀏覽器暫存留存資料
（供下次開啟還原）		 提供「清除暫存資料」隨時清除；發信紀錄設上限並可清空； 不使用小幫手時可於功能開關停用整個模組 已控制

5.3 適用邊界

適用個人專屬之公務電腦，且該電腦已納入公司端點管理與磁碟加密政策
不適用共用帳號環境、公用電腦、跨部門共享之工作站； 如須多人共用檢視，應改以具身分驗證之伺服器型系統實作

5.4 需風險承擔事項

RA-01　瀏覽器暫存於同一 Windows 帳戶內可被讀取
匯入之內容以 Base64（編碼而非加密）存於 localStorage 供還原使用， 同一帳戶下之其他程式可讀取。應用層無法消除此性質——加密金鑰須置於前端，等同鎖與鑰匙同放。
影響範圍：限於該 Windows 帳戶內；跨帳戶無法讀取；不涉及網路外洩。
補償措施：提供「清除暫存資料」功能供使用後清除；依公司磁碟加密政策保護靜態資料； 限定於個人專屬電腦使用。
風險水準：與該帳戶下其他本機檔案（含來源 Excel 本身）同等，無額外惡化。

5.5 系統定位聲明

本工具為弱點報告之輔助檢視與追蹤介面，非資料權威來源。 統計結果供追蹤管理使用，正式陳報與稽核應以原始弱點報告為準。
為利核對，所有統計數字均可下鑽至明細逐筆查證；畫面固定顯示目前母體條件 （部門／結案狀態／筆數）；欄位或狀態無法辨識時主動警示。

6　驗證方式與結果

6.1 驗證方法

項目方式
功能驗證（黑箱）以測試資料實際操作，逐一比對統計數字與明細筆數（50 項）、各功能頁面與下鑽行為
程式碼審查（白箱）全部程式碼逐行檢視，涵蓋邊界條件、錯誤處理、狀態管理與存取控制
存取控制驗證實測未帶權杖、帶錯誤權杖、帶正確權杖之回應行為
回歸驗證每次修正後重新執行完整測試，確認未衍生新問題

6.2 結果

  • 列管問題 35 項，已全數修正完畢（含 1 項存取控制缺失、多項資料正確性與催辦可靠性問題）
  • 存取控制實測：未帶權杖與帶錯誤權杖均正確拒絕（HTTP 401），帶正確權杖始放行
  • 最終回歸：全模組正常載入、各功能頁面正常、統計與明細比對 50/50 相符、無執行期錯誤
上述問題均為發布前之內部自我檢測所發現，於開放使用前完成修正，未曾造成實際作業影響。 詳細清單與修正對照留存於內部測試報告。

6.3 後續建議

  • 補強自動化測試（日期解析、狀態分類、數字對帳三項），降低後續修改之回歸風險
  • 依部門試用回饋持續調整；如納入正式作業流程，另依規定辦理相關程序

7　使用者責任

部分風險無法由系統控制，須由使用行為配合。

  • 原始報告保管：含真實主機與弱點細節之 Excel 屬機敏資料，不得任意轉寄或置於共用空間
  • 使用後清除：於共用或非個人專屬電腦使用後，須執行「其他功能 → 清除暫存資料」
  • 紀錄保管：匯出之發信紀錄含同仁電子郵件，須比照個人資料保管，不得外流
  • 權杖保管：agent_token.txt 等同寄信授權，不得外傳或置於共用位置
  • 異常回報：發現統計異常、寄送異常或疑似安全問題，請即聯繫系統維護窗口，勿自行修改程式檔案

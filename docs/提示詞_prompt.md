# 弱點彙總 Dashboard — 可複製提示詞（Prompt）

> 用途：把下面整段貼給 AI（Claude / ChatGPT 等），即可產生一套類似的弱點追蹤看板工具。
> 建議搭配：附上你的 Excel 欄位清單、或一份去識別化的樣本檔，效果最好。

---

## 一、核心提示詞（整段複製）

```
【角色】
你是資深前端工程師。請幫我做一套「弱點彙總 Dashboard」工具：讀取弱點掃描 Excel，
自動產生可互動的 HTML 追蹤看板。全程繁體中文。

【技術限制】
- 純前端：HTML + CSS + 原生 JavaScript，函式庫只用 SheetJS(xlsx) 與 Chart.js。
- 純本機解析：Excel 只在瀏覽器讀取，不上傳任何伺服器；免安裝、可離線。
- 分檔輸出完整程式，不得用 TODO 或「省略」，不得省略任何程式碼。
- 專案結構：index.html、css/、js/（依功能拆多支）、config/、assets/。

【Excel 來源】
- 工作表名稱：1-系統弱點掃描弱點（找不到時退用第一個工作表）。
- 欄位（依「名稱」讀取，不依位置；欄位順序改變仍須正常）：
  Plugin ID, Risk, Severity, Host, Protocol, Port, Name, Synopsis, Description,
  Solution, See Also, Plugin Output, 修補期限, 首次展延上限, 例外核准期限, 逾期狀態,
  系統類別, 資產名稱, 負責單位, 負責人, 複測狀態, 結案狀態, 結案日期, 備註, 年度, 純期系統
- 相容性：
  - 缺少必要欄位時，畫面明確提示「缺哪些欄位」＋列出實際讀到的標題，不得當機。
  - 需容錯「合併欄」：若 Risk 與 Severity 合併成單一欄（如「Risk Severity」/「Risk/Severity」），
    自動辨識並同時當作 Risk 與 Severity。
  - 欄位別名、分析條件、門檻集中在 config，方便調整。

【分析規則】
- 分析母體：負責單位 =「資訊架構部」且 結案狀態 =「未結案」（逐字比對、去空白）。
- 真正到期日：例外核准期限 → 首次展延上限 → 修補期限（依序取第一個有值者）。
- 逾期天數 = 今日 − 真正到期日（正值為已逾期）。
- 到期分類一律用「互斥區間」，不要用累進，讓各區間相加剛好＝未結案，可驗算：
  已逾期 / 30天內 / 31–90天 / 91–180天 / 180天以上 / 無到期日。

【功能：分四個分頁】
TAB1 Dashboard（主管總覽）
- 指標卡分「三個群組框」：
  (1) 總覽：未結案總數
  (2) 到期時間帶（互斥）：已逾期/30天內/31–90天/91–180天/180天以上/無到期日
  (3) 嚴重度：Critical/High/Medium/Low
  每張卡可點擊 drill-down。
- 「嚴重度 × 修復狀態」表：各嚴重度的 未修復(未結案) / 已修復(結案) / 合計 / 修復率(進度條)。
- 今日行動清單：逾期＋今日到期＋例外即將到期，依風險分排序。
- 風險加權排行 Top N：分數 = 嚴重度權重 × 急迫倍數（逾期越久越高）。
- 圖表：嚴重度圓餅、到期時間帶長條（互斥）。

TAB2 人員追蹤（催辦）
- 每位負責人一列：未結案、已逾期、30天內、31–90天、91–180天、180天以上、無到期日
  （互斥，各列相加＝該人未結案），合計列相加＝總未結案。數字可 drill-down。
- 每人「產生催辦內容」：自動列出該人「已逾期＋快到期」的弱點，湊成可複製 Mail（主旨＋內文），
  可一鍵複製 / 開啟郵件軟體；另提供「批次產生全部催辦」。
- 「未來六個月到期矩陣（人員 × 月）」：每人各月到期筆數＋合計，格子可點開明細。

TAB3 例外/展延統計（治理）
- 依「處置階段」分類：例外管理中（有例外核准）/ 首次展延中（有展延無例外）/ 原始修補期限 / 未定期限。
  各階段：件數、占比、已逾期、快到期、尚安全，可 drill-down。
- 安全名單：例外管理中且例外核准期限尚未到 → 暫免催辦。
- 例外治理雷達：例外覆蓋率、例外破口（例外中卻已逾期）、例外到期預警(7/14/30天分級)、
  慢性風險（同一弱點反覆展延/例外 ≥ 2 次，靠備註中的 iForm 次數判斷）。
- 圖表：處置階段占比圓餅、到期時間帶 × 階段 堆疊長條。

TAB4 查詢
- 全文搜尋（空格分隔多字 AND；跨主機/負責人/弱點名/Plugin ID/備註/日期）。
- 快速篩選 chips：已逾期 / 快到期 / 六個月到期 / 各嚴重度 / 例外中 / 首次展延中 / 安全名單 / 慢性風險。
- 結果用可排序明細表呈現，可匯出 CSV、另開新分頁。

【通用 UX】
- 所有表格「點欄位標題可排序」（升/降序切換，空值一律墊底）。
- drill-down 明細欄位：Host、負責人、Name、Risk、Severity、Plugin ID、修補期限、首次展延上限、
  例外核准期限、真正到期日、逾期天數、備註。
- 明細表：固定高度捲動視窗（左右捲軸永遠在視野內、不用捲到最底）、凍結第一欄。
- 「另開新分頁」：把任一明細彈出成獨立可排序頁面，方便多張並排比對。
- 匯出 CSV（含 UTF-8 BOM 讓 Excel 正確辨識）。
- 母體對帳可顯示「未結案 + 結案」兩總（資訊架構部 = 未結案 + 結案 + 其他）。

【記住上次匯入】
- 匯入成功後用 localStorage 存起來（只存有用到的欄位以縮小體積；日期存成 YYYY/M/D 純文字避免時區位移）。
- 下次開啟自動還原上次資料，不必重匯；匯入新檔才覆蓋。
- 提供「換一個檔案」與「🗑 清除記憶」按鈕。

【動手前】
先用一兩句列出你對需求的理解與關鍵假設，讓我確認後再開始；然後分批交付完整程式。
```

## 二、加速迭代的小技巧（給模仿者）

1. **先給資料樣貌**：把你的 Excel「標題列 + 2~3 筆去識別化資料」貼給 AI，能大幅減少來回。
2. **一次一個焦點**：先做出能跑的版本，再逐項加（催辦 → 統計 → 查詢 → 記憶…），不要一次塞爆。
3. **要求可驗算**：明講「分類要互斥、相加要等於總數」，AI 就會自己附對帳，減少算錯。
4. **要求先問後做**：加一句「動手前先列理解與假設」，避免它憑空猜、你踩雷。
5. **實測回饋**：把畫面截圖或錯誤訊息貼回去，讓 AI 依實況修（例如欄位合併、數字對不上）。
6. **明確禁止省略**：「不得用 TODO/省略、分檔完整輸出」——否則常拿到半成品。

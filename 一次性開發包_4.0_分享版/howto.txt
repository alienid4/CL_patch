pipeline/               ← 引擎：直接複製到你專案的 .project/
  ├ loop.py             ← 現成的自主驅動器，複製過去就能跑，不用 AI 寫
  ├ checks.py           ← 現成的關卡腳本
  ├ snapshot.py          ← 現成的看現況腳本
  ├ loop_config.json     ← 煞車設定範本，你要自己填一行
  ├ decisions.json       ← 空白範本
  └ backlog.json         ← 待辦清單範本
一次性開發提示詞_4.0_FINAL.md   ← 這份才是要貼給 AI 的，是「行為規則」不是「叫它寫 loop.py」

別人三步驟（SETUP.md 寫的）：

把 pipeline/ 整包複製進對方專案的 .project/，git init
把 一次性開發提示詞_4.0_FINAL.md 貼給對方的 AI（Claude Code 或 Codex），AI 讀了就知道要照 4.0 紀律做事——但這是叫 AI「照規矩做功能」，不是叫它「寫 loop.py」
對方自己編輯 loop_config.json 填一行 agent_cmd（例如 "claude -p --permission-mode acceptEdits"），指定 loop.py 要呼叫哪個 AI CLI
所以是的，包給別人基本上就能用，但有 3 個前提要提醒對方：

對方自己要有裝好、登入好的 Claude Code 或 Codex CLI（包裡沒附 AI 本身，只附紀律骨架）
包裡有一句誠實警語：「loop.py 是就緒版，建議先跑一次驗證再依賴」——要先 --dry-run 再 --once 測過，看到「你沒插手、它自己做完並 commit」才算真的驗證成功
對外分享前要先掃一遍 方法規則/ 資料夾，確認沒有殘留你公司/專案名稱——這點包裡自己也提醒了，你這次要送出去之前務必檢查一下

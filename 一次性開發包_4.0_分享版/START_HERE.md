# 一次性開發包 4.0 — 從這裡開始

一包「讓 AI 自主開發、但有護欄」的方法 + 工具。丟進任何專案就能用。

## 這包裡有什麼

```
一次性開發提示詞_4.0_FINAL.md   ← 主提示詞（貼給 AI，它照這個工作）
CHANGELOG.md                     ← 版本演進
方法規則/                        ← 各種 gate / 安全 / 驗證規則（提示詞會用到）
pipeline/                        ← 引擎：放進你專案的 .project/
  ├ snapshot.py        看現況
  ├ checks.py          強制關卡（可攜版：沒測試會警告放行）
  ├ loop.py            自主驅動器（按一次、走開）
  ├ loop_config.json   煞車設定 + 你的 AI CLI
  ├ decisions.json     重大事項紀錄（空的）
  └ backlog.json       待辦清單（範本）
SETUP.md                         ← 3 步安裝
```

## 三步上手

1. **安裝**：照 `SETUP.md` 把 `pipeline/` 放進你專案的 `.project/`，並 `git init`。
2. **讀方法**：把 `一次性開發提示詞_4.0_FINAL.md` 貼給你的 AI（Claude Code / Codex 皆可），它就進入「4.0 自主模式」。
3. **出題 + 走開**：用下面啟動語，把你要做的填進去。

### 啟動語（貼給 AI）

```
請讀 一次性開發提示詞_4.0_FINAL.md，照它進入「4.0 自主模式」。
我要做：<一句話講你要做的功能/系統>
新系統先訪談我確認 MVP；小切片、每件自己跑 python .project/checks.py 驗過再 commit。
碰到金流/權限/刪資料/正式機/語意不明就停下問我。
設好回合與時間上限，到頂出待續報告。我走開，做完或卡住再回來。
```

## 核心概念（30 秒）

- **你出題 + 驗收**：定義「要什麼、怎樣算對」；AI 負責「怎麼做」。
- **有護欄才敢放手**：checks 過不了擋、碰大事停下問人、卡住/到預算自己停。
- **升一級，別跳頂**：先學「交辦」，再談「並行」。

## ⚠️ 誠實標示

- `loop.py`（全自動驅動器）是**就緒版，建議先跑一次驗證**再依賴它（見 SETUP.md 的驗證步驟）。在它驗證前，可先用「主提示詞 + 你手動觸發」達到 4.0 行為。
- 若要**對外分享**：先掃一遍 `方法規則/` 有沒有殘留你公司/專案名稱再送出。

## 1. 這是什麼專案（一句話）

讀「弱點彙總報告」Excel，在瀏覽器本機產生弱點追蹤 Dashboard。**純前端**（SheetJS + Chart.js），
不上傳、免安裝。近期在做**多工作表擴充**：左側 10 張數字表導覽，每張一套看板。

- 進入點：`index.html`（雙擊或用 `py -m http.server` 開）
- 檔案地圖：`config/`（設定/欄位對應）、`js/`（邏輯，依功能分檔）、`css/`、`docs/`（說明/對照表/檢查頁）
- 多表核心：`config/profiles.js`（欄位對應）+ `js/sheets.js`（讀取正規化）+ `js/multi.js`（組看板 result）
- 檢查工具：`docs/多表檢查.html`（丟 Excel 看各表對映對不對，自帶邏輯、雙擊即可用）

**js 模組地圖（依功能分檔，全域掛在 window.*）：**
- `main.js` 主控（上傳/解析/分頁切換/左側導覽/全域篩選 state）｜`analysis.js` 核心分析（BANDS/bandOf/severity/assembleResult…）｜`multi.js` 多表組裝
- `dashboard.js` 各項目看板（指標卡/各嚴重度結案進度/優先處理清單/風險排序/圖表）｜`tracking.js` 人員追蹤｜`stats.js` 例外/展延統計
- `matrix.js` 交叉分析（嚴重度×到期，排除式漏斗）｜`summary.js` 主管總覽首頁（KPI/項目表/趨勢/SLA/紅黑榜）｜`history.js` 歷史快照＋趨勢
- `search.js` 查詢｜`reminder.js`+`email.js` 催辦/Email｜`ui-common.js` 共用（Modal/明細表/drill/toast/CSV/`UI.drillEvents` 圖表鑽取）
- `features.js` 功能開關（讀 `config/features.js` 註冊表）

## 2. 接手第一步（照順序）

1. `py .project/snapshot.py` → 生成/更新 `CURRENT_STATE.md`（HEAD、checks、決策、**待辦＋下一件**、**worklog 尾巴**、近 10 commit）
2. `py .project/checks.py` → 應「全部 PASS」
3. 讀 `CURRENT_STATE.md`（尤其 **worklog 尾巴** = 上次斷在哪）、本檔、`.project/decisions.json`、`.project/多表結構_快取.md`
4. `git branch` / `git log --oneline` → 確認在哪個分支、做到哪
   - **現況：多表擴充早已合併，近期(V1.08+)都直接在 `main` 上開發**（單人快速迭代，每版 bump `config/version.js` + commit）。`work/multi-sheet` 是歷史分支。

## 3. 續作 SOP（4.0）

- 從 `.project/backlog.json` 拉**下一件 `todo`**；切 `work/<名稱>` 分支（**別在 main 上做**）
- 拆小切片，一件只改一塊 → 自驗 `py .project/checks.py` 過 **＋** 用 `docs/多表檢查.html` 或預覽抽樣對帳 → `git commit`（訊息一句人話）
- 做完把該 backlog 項目 `status` 改 `done`
- **絆線就停下問人並寫 `decisions.json`**：金流/權限/刪資料/改設定檔/正式機/語意不明/同一件卡 3 次
- **⚠ 別過度確認**：可逆的低風險操作（本機 `git merge`/`git tag`/切片開做/小改）**自己做，不要一直回頭問 USER「要不要」「go?」**。4.0 的預設是自主，問太多＝違反 4.0。
- **版本標籤慣例**：每個版本 bump `config/version.js` 後，打對應 git tag（`v1.0`、`v1.01`…）當還原點。USER 說「退回 vX」就 `git checkout`/`git reset` 到該 tag。

### 防「隨時斷線」的紀律（Wi-Fi 隨時可能斷 → 半成品要救得回）
- **邊做邊寫 `.project/worklog.md`**：開切片前先寫「要做/怎驗/步驟」，過程中更新「做到哪/剛發現」，**存檔要勤**（硬斷線只有磁碟上的活得下來）。
- **勤 commit**：在 `work/` 分支上，做一半也可先 `commit`（訊息前綴 `WIP:`）。切片切更小 → 每次斷線最多丟最後幾分鐘。
- **斷線回來第一件事**：`py .project/snapshot.py` → 讀 `CURRENT_STATE.md` 最上面的 **worklog 尾巴**，就知道接哪一步；別重讀整包程式猜。
- **不用重撈資料**：Excel 結構/欄位/筆數/雷 都在 `.project/多表結構_快取.md`，直接讀，別重開 Excel 掃。

## 5. 踩雷筆記（真的踩過，別再踩）

- **`python` 在這台是商店空殼**（exit 49、無輸出）→ 一律用 **`py`** 跑 .py。
- **Excel 檔常被開著鎖住**（openpyxl/瀏覽器讀不到、PermissionError）→ 先關 Excel，或先 `cp` 一份再讀。
- **檔案在 OneDrive**：有些 `.js` 可能是「雲端未下載」佔位檔，`file://` 跨目錄載入會失敗 → `docs/多表檢查.html` 已改成**自帶邏輯**（不依賴 ../js）。
- **記憶（localStorage）**：多表版存「整份檔 base64 + 目前選的表」（key `vulnDashboard.v2`），重開自動還原。
- **git 的 `LF will be replaced by CRLF` 警告無害**，不用理。
- **這份 TEST 檔是稀疏樣本**：表 1/2/3/4 只有 1 筆且多為已結案 → 未結案顯示 0 是正常，換完整資料檔就有數字。

## 7. 近期架構（V1.08+，穩定說明）

**產品定位＝雙軌**：窗口做事（各項目看板、催辦、交叉分析） + 主管稽核（總覽首頁、紅黑榜、SLA、趨勢、全域逾期）。

### 7.1 功能開關框架（模組化，A＋B 兩層）— V1.22
- **layer A 出廠預設**：`config/features.js` 的 `window.APP_FEATURES` 註冊表（隨資料夾發布，主管定調全體）。
- **layer B 個人覆寫**：`js/features.js` `Features.set/isOn`，存 `localStorage『vulnDashboard.features』`。
- `Features.isOn(id)` = 個人有覆寫用覆寫，否則吃 config 預設。UI 在「其他功能 → 功能開關」(`#features-btn`)。
- **加新功能只要 2 步**：① `config/features.js` 加一行 `{id,group,label,default}`；② 渲染處用 `Features.isOn('id')` 包起來。設定面板會自動長出勾選格。
- 分頁隱藏統一走 `main.js applyTabVisibility(caps)`（caps 面板自適應 + 功能開關，任一不通就藏，當前分頁被藏退回總覽）。

### 7.2 三個主管模組（都在總覽首頁 summary.js，都可開關）
- **紅黑榜** `panel-red-list`（V1.23）：跨全部項目依部門/負責人排名(逾期多者在前)，紅列=有逾期、綠列=全結案；`makeSortableTable` 通用可排序表，**預設前 5 名＋展開全部**(limit)；隨左側「部門」選擇器縮。
- **SLA 達成率** `panel-sla`（V1.24）：政策天數在 `config.sla`(Critical7/High30/Medium90/Low180，可改)；因**資料無發現日**，達成率=各嚴重度未結案中未逾期比率(非真時效)，日後有發現日再升級。
- **趨勢(跟上次比)** `panel-trend`（V1.25）：`js/history.js` 每次**真實匯入(handleFile)**自動記輕量快照(彙總數字+未結弱點指紋 host||pluginId)，同檔同日去重，`localStorage『vulnDashboard.history』`留最近12期；**範例載入/自動還原不記**。本次vs上次(新增/結案消失/淨變化，可點) + 未結/逾期趨勢線。

### 7.3 其他近期重點
- **全站可鑽取**（V1.18）：所有圖表(圓餅/長條/堆疊)＋百分比都可點看實際筆數，共用 `UI.drillEvents()`。
- **交叉分析**（V1.15–19）：`matrix.js` 嚴重度×到期矩陣，**排除式**(點列首/欄首=把那個藏起來，其餘留)＋部門/負責人 facet(聚焦)＋格子數字 drill。
- **全域結案狀態**（V1.20）：結案狀態選擇器移到左側「部門」下面，全站持久化(`vulnDashboard.close`)，進項目不重設。
- **使用者長期偏好**：畫面**不放說明性/備註性文字、不放括號註解、不放教學 tooltip**（像 Excel 點每格都沒備註）。新增 UI 一律遵守。
- **localStorage keys 總覽**：`vulnDashboard.v2`(整份檔+選的表)、`.dept`(我的部門)、`.close`(結案狀態)、`.features`(功能開關覆寫)、`.history`(歷史快照)。
